# 机型动态参数调整功能

## 功能概述

Lumen 现在支持根据运行设备的硬件配置自动调整 AI 模型参数，以优化性能和资源使用。系统会自动检测 CPU 架构、内存大小等信息，并应用最佳的 GPU 加速层数和上下文大小配置。

## 核心特性

### 1. 自动机型检测

系统在启动时会自动检测以下硬件信息：
- CPU 架构（Apple Silicon vs Intel）
- 内存大小
- CPU 核心数
- 具体型号（通过 sysctl 查询）

### 2. 智能参数推荐

根据不同机型档次，系统会自动配置：

#### Apple Silicon Max/Ultra (M1/M2/M3 Max, 64GB+ RAM)
```typescript
{
  gpuLayers: 99,        // 尽可能使用 GPU
  contextSize: 8192,    // 最大上下文
  temperature: 0.7,
  topP: 0.9
}
```

#### Apple Silicon Pro (M1/M2/M3 Pro, 32GB+ RAM)
```typescript
{
  gpuLayers: 64,
  contextSize: 6144,
  temperature: 0.7,
  topP: 0.9
}
```

#### Apple Silicon Base (M1/M2/M3, <32GB RAM)
```typescript
{
  gpuLayers: 32,
  contextSize: 4096,
  temperature: 0.7,
  topP: 0.9
}
```

#### Intel Mac 高配 (32GB+ RAM)
```typescript
{
  gpuLayers: 48,
  contextSize: 4096,
  temperature: 0.7,
  topP: 0.9
}
```

#### Intel Mac 低配 (<32GB RAM)
```typescript
{
  gpuLayers: 16,
  contextSize: 3072,
  temperature: 0.7,
  topP: 0.9
}
```

### 3. Embedding 模型优化

Embedding 模型的配置会根据机型自动调整：
- **Apple Silicon**: 默认使用 GPU 加速（99 层）
- **Intel Mac**: 根据配置使用 0-24 层 GPU
- **基础机型**: 使用 CPU 以保证稳定性

## 使用方法

### 默认行为（自动推荐）

什么都不用做！系统会在启动时自动检测并应用最佳配置。

```typescript
// LumenCore 初始化时会自动检测
const lumen = new LumenCore();
await lumen.initEngine();
// 系统已根据机型自动配置参数
```

### 查看当前配置

```typescript
const selection = await window.api.getCurrentModelSelection();
console.log(selection);
// {
//   llm: "Qwen3.5-4B-Q4_K_M.gguf",
//   embedding: "bge-m3-q8_0.gguf",
//   llmGpuLayers: -1,      // -1 表示自动
//   embeddingGpuLayers: -1,
//   contextSize: -1
// }
```

### 手动覆盖参数

如果需要手动调整，可以传递 `null` 以外的值来覆盖自动推荐：

```typescript
// 手动设置特定参数
await window.api.setModels({
  llmModelFile: 'Qwen3.5-4B-Q4_K_M.gguf',
  llmGpuLayers: 50,      // 自定义 GPU 层数
  contextSize: 4096,     // 自定义上下文大小
});

// 使用自动推荐（设置为 null）
await window.api.setModels({
  llmModelFile: 'Qwen3.5-4B-Q4_K_M.gguf',
  llmGpuLayers: null,    // null 表示使用自动推荐
  contextSize: null,
});
```

### 混合配置

可以部分使用自动推荐，部分手动设置：

```typescript
await window.api.setModels({
  llmModelFile: 'custom-model.gguf',
  llmGpuLayers: 40,      // 手动设置
  contextSize: null,     // 自动推荐
  embeddingGpuLayers: null, // 自动推荐
});
```

## 技术实现

### SystemMonitor 类

核心监控类，提供以下功能：

```typescript
import { systemMonitor } from './tools/system_monitor';

// 获取系统信息
const info = await systemMonitor.getSystemInfo();
// {
//   platform: "darwin",
//   arch: "arm64",
//   cpuCores: 10,
//   totalMemoryGB: 64,
//   isAppleSilicon: true,
//   machineType: "apple_silicon_high"
// }

// 获取推荐的 LLM 参数
const llmParams = await systemMonitor.getRecommendedLLMParams();

// 获取推荐的 Embedding 参数
const embeddingParams = await systemMonitor.getRecommendedEmbeddingParams();

// 清除缓存（重新检测）
systemMonitor.clearCache();
```

### 机型判断逻辑

1. **CPU 架构检测**: 通过 `uname -m` 判断是 arm64 还是 x86_64
2. **CPU 型号检测**: 通过 `sysctl -n machdep.cpu.brand_string` 获取详细型号
3. **内存检测**: 通过 `os.totalmem()` 获取总内存
4. **综合判断**: 结合以上信息确定机型档次

### 参数继承机制

在 `LLMEngine` 内部，配置参数遵循以下优先级：

```typescript
// 方法调用时的选项 > 初始化时的配置 > 默认值
async streamChat(prompt, onToken, options) {
  await this.session.prompt(prompt, {
    temperature: options?.temperature ?? this.config.temperature ?? 0.7,
    topP: options?.topP ?? this.config.topP ?? 0.9,
  });
}
```

## 故障排除

### 问题：系统检测失败

**症状**: 日志显示 "unknown" 机型

**解决方案**:
1. 检查系统权限是否允许执行 `sysctl` 命令
2. 查看错误日志获取详细信息
3. 系统会降级到保守配置保证可用性

### 问题：性能不如预期

**症状**: 推理速度慢或显存占用过高

**解决方案**:
1. 尝试手动降低 `gpuLayers` 值
2. 减小 `contextSize` 配置
3. 查看日志确认实际应用的参数

### 问题：手动设置不生效

**症状**: 调用 `setModels` 后参数未变化

**解决方案**:
1. 确保传递正确的参数类型（`number | null`）
2. 检查返回值是否为 `-1`（表示自动推荐）
3. 查看日志确认模型重新加载成功

## 注意事项

1. **首次启动**: 系统会在首次启动时检测机型，结果会被缓存
2. **缓存清除**: 更换硬件后可调用 `clearCache()` 重新检测
3. **降级方案**: 如果检测失败，系统会使用保守配置确保稳定
4. **日志记录**: 所有参数调整都会记录到日志中便于调试

## API 参考

### ElectronAPI 接口

```typescript
interface ElectronAPI {
  // 获取当前模型配置（-1 表示自动）
  getCurrentModelSelection: () => Promise<{
    llm: string;
    embedding: string;
    llmGpuLayers: number;
    embeddingGpuLayers: number;
    contextSize: number;
  }>;

  // 切换模型（支持 null 表示自动）
  setModels: (options: {
    llmModelFile?: string;
    embeddingModelFile?: string;
    llmGpuLayers?: number | null;
    embeddingGpuLayers?: number | null;
    contextSize?: number | null;
  }) => Promise<boolean>;
}
```

### LLMModelConfig 接口

```typescript
interface LLMModelConfig {
  modelPath?: string;
  contextSize?: number;
  temperature?: number;      // 生成多样性 (0.0-1.0)
  topP?: number;             // 核采样参数 (0.0-1.0)
  gpuLayers?: number | null; // GPU 层数，null 为自动
}
```
