# 跨进程日志系统使用指南

## 📋 概述

本项目实现了完整的跨进程日志系统，允许渲染进程（Renderer）通过 IPC 将日志发送到主进程（Main），由主进程的 `electron-log` 统一管理并写入文件。

## 🏗️ 架构设计

```
┌─────────────────────┐
│  Renderer Process   │
│  (React Frontend)   │
│                     │
│  window.logger      │
│  ↓                  │
│  preload/index.ts   │
└─────────┬───────────┘
          │ IPC (异步)
          ↓
┌─────────────────────┐
│   Main Process      │
│  (Electron Backend) │
│                     │
│  ipcMain handlers   │
│  ↓                  │
│  LoggerService      │
│  ↓                  │
│  electron-log       │
│  ↓                  │
│  日志文件            │
└─────────────────────┘
```

### 2. Preload 层 - API 暴露

**文件**: `src/preload/index.ts`

Preload 脚本通过 `contextBridge` 向渲染进程暴露 `window.logger` 对象：

```typescript
// 定义 Logger 接口
interface Logger {
  info(message: string, context?: string, data?: any): void;
  warn(message: string, context?: string, data?: any): void;
  error(message: string, context?: string, data?: any): void;
  debug(message: string, context?: string, data?: any): void;
}

// 创建日志发送函数（使用异步 IPC）
const createLogFunction = (level: LogLevel) => {
  return (message: string, context?: string, data?: any) => {
    setImmediate(() => {
      ipcRenderer.send(IPC_LOG_ACTION, {
        level,
        message,
        source: LogSource.Renderer,
        context: context || 'RENDERER',
        timestamp: new Date().toISOString(),
        data,
      });
    });
  };
};

// 暴露到渲染进程
contextBridge.exposeInMainWorld('logger', {
  info: createLogFunction(LogLevel.Info),
  warn: createLogFunction(LogLevel.Warn),
  error: createLogFunction(LogLevel.Error),
  debug: createLogFunction(LogLevel.Debug),
});
```

**关键特性**:
- ✅ 使用 `setImmediate` 异步发送，不阻塞渲染进程主线程
- ✅ 自动添加时间戳和来源标识
- ✅ 类型安全的接口定义

### 3. Main Process - 日志接收与转发

**文件**: `src/main/ipc/handlers.ts`

```typescript
import { logger } from '@main/tools/logger';
import { LogPayload, LogSource } from '@shared/types';

// 监听来自渲染进程的日志请求
ipcMain.on(IPC_LOG_ACTION, (event, payload: LogPayload) => {
  // 验证日志来源
  if (payload.source !== LogSource.Renderer) {
    return;
  }

  // 构建带前缀的消息格式
  const prefix = '[Renderer]';
  const context = payload.context ? `${prefix}:${payload.context}` : prefix;

  // 根据日志级别调用对应的 Logger 方法
  switch (payload.level) {
    case 'debug':
      logger.info(payload.message, context); // debug 映射到 info
      break;
    case 'info':
      logger.info(payload.message, context);
      break;
    case 'warn':
      logger.warn(payload.message, context);
      break;
    case 'error':
      logger.error(payload.message, payload.data); // 传递错误对象
      break;
    default:
      logger.info(payload.message, context);
  }
});
```

**关键特性**:
- ✅ 自动添加 `[Renderer]` 前缀，便于区分日志来源
- ✅ 保留原始日志级别信息
- ✅ 复用主进程现有的 Logger 实例

### 4. Renderer 层 - 统一封装（推荐）

**文件**: `src/renderer/tools/logger.ts`

提供了一个更高级的封装，支持在 React 组件和 Zustand Store 中直接使用：

```typescript
import { LogLevel } from '@shared/types';

// 全局 logger 实例
export const logger = {
  info(message: string, context?: string, data?: any) {
    // 通过 window.logger 发送到主进程
    if (typeof window !== 'undefined' && window.logger) {
      window.logger.info(message, context, data);
    }
  },
  warn(message: string, context?: string, data?: any) { /* ... */ },
  error(message: string, context?: string, data?: any) { /* ... */ },
  debug(message: string, context?: string, data?: any) { /* ... */ },
};

// 创建带上下文的 logger 实例
export function createLogger(defaultContext: string) {
  return {
    info: (message: string, data?: any) => logger.info(message, defaultContext, data),
    warn: (message: string, data?: any) => logger.warn(message, defaultContext, data),
    error: (message: string, data?: any) => logger.error(message, defaultContext, data),
    debug: (message: string, data?: any) => logger.debug(message, defaultContext, data),
  };
}
```

## 💡 使用示例

### 在 React 组件中使用

```tsx
import React, { useEffect, useState } from 'react';
import { logger } from '@renderer/tools/logger';

const StockPanel: React.FC = () => {
  const [stocks, setStocks] = useState([]);

  useEffect(() => {
    logger.info('StockPanel 组件已挂载', 'Component');
    
    fetchStocks().catch((error) => {
      logger.error('获取股票数据失败', 'StockPanel', error);
    });

    return () => {
      logger.info('StockPanel 组件已卸载', 'Component');
    };
  }, []);

  const fetchStocks = async () => {
    logger.info('开始获取股票数据', 'API');
    try {
      const response = await fetch('/api/stocks');
      const data = await response.json();
      setStocks(data);
      logger.info(`成功获取 ${data.length} 只股票`, 'API');
    } catch (error) {
      logger.error('股票数据获取失败', 'API', error);
      throw error;
    }
  };

  return <div>...</div>;
};
```

### 在 Zustand Store 中使用

```typescript
import { create } from 'zustand';
import { logger, createLogger } from '@renderer/tools/logger';

// 方式 1: 直接使用全局 logger
interface ChatStore {
  messages: Message[];
  sendMessage: (content: string) => Promise<void>;
}

export const useChatStore = create<ChatStore>((set, get) => ({
  messages: [],
  
  sendMessage: async (content: string) => {
    logger.info(`用户发送消息：${content.substring(0, 50)}...`, 'ChatStore');
    
    try {
      const result = await window.lumen_core.sendMessage(content);
      
      set((state) => ({
        messages: [...state.messages, { role: 'user', content }],
      }));
      
      logger.info('消息发送成功', 'ChatStore');
    } catch (error) {
      logger.error('消息发送失败', 'ChatStore', error);
      throw error;
    }
  },
}));

// 方式 2: 创建带上下文的 logger（更简洁）
const apiLogger = createLogger('API');

interface DataStore {
  fetchData: () => Promise<void>;
}

export const useDataStore = create<DataStore>((set) => ({
  data: null,
  
  fetchData: async () => {
    apiLogger.info('开始获取数据');
    
    try {
      const response = await fetch('/api/data');
      const data = await response.json();
      set({ data });
      apiLogger.info('数据获取成功', { count: data.length });
    } catch (error) {
      apiLogger.error('数据获取失败', error);
      throw error;
    }
  },
}));
```

### 在工具函数中使用

```typescript
// src/renderer/tools/api.ts
import { createLogger } from '@renderer/tools/logger';

const apiLogger = createLogger('API');

export async function request<T>(url: string, options?: RequestInit): Promise<T> {
  apiLogger.info(`请求：${url}`, { method: options?.method || 'GET' });
  
  const startTime = Date.now();
  
  try {
    const response = await fetch(url, options);
    
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }
    
    const data = await response.json();
    const duration = Date.now() - startTime;
    
    apiLogger.info(`响应：${url}`, { status: response.status, duration: `${duration}ms` });
    return data;
  } catch (error) {
    apiLogger.error(`请求失败：${url}`, error);
    throw error;
  }
}
```

## ⚙️ 配置说明

### 日志文件路径

日志文件默认保存在系统日志目录：
- **macOS**: `~/Library/Logs/lumen-YYYY-MM-DD.log`
- **Windows**: `%APPDATA%\lumen\logs\lumen-YYYY-MM-DD.log`
- **Linux**: `~/.config/lumen/logs/lumen-YYYY-MM-DD.log`

### 日志清理策略

主进程的 `LoggerService` 会自动清理 7 天前的日志文件：

```typescript
// 在 LumenCore 初始化时调用
logger.pruneOldLogs(7); // 保留最近 7 天的日志
```

### 日志级别控制

当前所有级别的日志都会记录和显示。如需过滤，可在主进程的 `logger.ts` 中调整：

```typescript
log.transports.file.level = "info"; // 可选：'debug' | 'info' | 'warn' | 'error'
```

## 🎯 最佳实践

### ✅ 推荐做法

1. **使用统一的 logger**: 始终通过 `@renderer/tools/logger` 导入 logger，不要直接使用 `console.log`
2. **提供有意义的上下文**: 
   ```typescript
   logger.info('操作成功', 'ModuleName'); // 而不是 logger.info('操作成功')
   ```
3. **记录关键错误**: 
   ```typescript
   try {
     // ...
   } catch (error) {
     logger.error('具体操作失败', 'ModuleName', error); // 传递 error 对象
   }
   ```
4. **使用 createLogger**: 为不同模块创建专用的 logger
   ```typescript
   const apiLogger = createLogger('API');
   const uiLogger = createLogger('UI');
   ```

### ❌ 避免的做法

1. 不要在循环中大量输出日志（会占用 IPC 带宽）
2. 不要记录敏感信息（密码、token 等）
3. 不要在日志中包含过大的数据对象

## 🐛 故障排查

### 问题：渲染进程的日志没有出现在日志文件中

**检查步骤**:
1. 打开开发者工具，确认 `window.logger` 存在
2. 检查控制台是否有 IPC 相关错误
3. 确认主进程的 IPC handler 已注册（查看 `src/main/ipc/handlers.ts`）
4. 检查日志文件路径是否正确

### 问题：日志延迟较高

**可能原因**:
- IPC 通信拥堵
- 主进程繁忙

**解决方案**:
- 减少不必要的日志输出
- 使用 `createLogger` 合并相同上下文的日志
